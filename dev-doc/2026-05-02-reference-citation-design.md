# 独立参考文献功能设计

## 背景

当前项目已经具备独立的文档上传、解析、预览与导出能力，降 AI 功能也已有完整工作流。本次新增功能不与降 AI 轮次耦合，而是作为一条独立文档处理流水线存在。

目标是在上传 `docx/txt` 后，系统能够联网检索与论文内容匹配的真实参考文献，在正文合适的句子位置自动插入 `[1]`、`[2]` 这类编号，并在文末生成仅被正文实际引用的参考文献列表。

## 目标

1. 支持独立的“添加参考文献”功能，不依赖现有降 AI 流程。
2. 参考文献必须来自真实来源，不能由模型虚构。
3. 支持用户指定中文文献数和英文文献数。
4. 正文引用标记应尽量落在对应句子后，而不是简单放在段尾。
5. 中文文献检索必须包含 Browser MCP + 用户登录知网流程后才能上线。
6. 搜索过程必须低频、可控，避免连续抓取、规避风控或处理敏感登录信息。

## 非目标

1. 不做全文 PDF 下载。
2. 不做大规模自动翻页抓取。
3. 不保存用户知网账号密码。
4. 不做绕过登录、绕过权限、绕过验证码或隐藏自动化识别的能力。
5. 第一版不追求学术写作意义上的“绝对正确引用”，但要通过真实来源、人工确认和规则约束把错误率压低。

## 推荐方案

采用双轨半自动引用工作台：

- 英文文献：后端直接接 `OpenAlex + Crossref`
- 中文文献：`Browser MCP + 用户手动登录知网 + 用户确认候选`
- 句级插入：系统自动推荐绑定，用户只确认候选文献，不手动逐句绑定

这是当前项目最适合的方案，因为它：

1. 保持现有 Flask + React + docx 流水线风格。
2. 把知网放在人工登录和人工确认的合规边界内。
3. 保留英文检索的自动化稳定性。
4. 允许系统完成句级绑定与 `[x]` 插入，避免用户负担过重。

## 总体流程

1. 用户进入“添加参考文献”工作台并上传 `docx/txt`
2. 后端解析正文结构，识别并排除已有参考文献区
3. 系统执行全文分析：
   - 识别章节
   - 切分段落和句子
   - 给句子打“引文需求分”
   - 聚类出有限数量的主题簇
   - 自动估算建议的中文/英文文献数
4. 用户确认或调整中文/英文文献目标数
5. 系统开始英文检索，得到英文候选文献
6. 系统进入知网辅助检索：
   - 打开知网页面
   - 用户自行登录
   - 系统按主题簇低频辅助搜索
   - 用户确认保留的中文候选文献
7. 系统合并中英文候选，自动生成句级绑定建议
8. 系统按首次出现顺序编号 `[1]...[N]`
9. 系统在句子位置插入引用编号，并在文末追加参考文献区
10. 用户预览结果并导出新的 `docx`

## 长文控制策略

不能对几万字论文逐句搜索，因此采用“先识别需求，再有限搜索”的策略。

### 引文需求识别

系统不直接对每句话检索，而是先识别高价值候选句，打分依据包括：

- 所在章节是否属于高引文密度区，如引言、文献综述、理论基础
- 句子是否包含事实性陈述、定义、已有研究、方法依据、研究结论
- 是否出现“已有研究表明”“相关学者认为”“根据某理论”等引文信号
- 句子是否已经带有引用标记

### 主题聚类

将高分句聚合成有限主题簇，而不是一条句子对应一次检索。

建议上限：

- 候选引文句：30 到 60 句
- 主题簇：10 到 20 个
- 每个主题簇的英文检索候选：3 到 8 条
- 中文知网检索词：5 到 10 个

### 文献复用

同一篇文献可以支撑多个相近句子。系统应支持：

- 一个主题簇复用 1 到 3 篇核心文献
- 同一段多个句子共享同一篇文献
- 避免在同段密集插入过多编号

## 真实可信约束

### 英文文献

英文文献必须通过真实数据库返回并至少满足：

- 标题
- 作者
- 年份
- 来源
- DOI 或可稳定识别的数据库 ID

检索后使用 `Crossref` 二次校验，完成去重和元数据补全。

### 中文文献

中文文献来自知网辅助检索流程，进入最终结果前至少应保留：

- 标题
- 作者
- 年份
- 来源
- 检索页面链接或可回溯定位信息

中文候选进入最终结果前必须经过用户确认。

## Browser MCP 与知网流程边界

知网能力必须设计成“人工登录、低频辅助、人工确认”，而不是自动抓取器。

### 允许的行为

1. 打开知网页面
2. 用户自行登录学校或个人账号
3. 按主题簇逐次输入搜索词
4. 读取当前页少量候选元数据
5. 由用户确认是否保留某条文献

### 禁止的行为

1. 批量连续翻页抓取
2. 批量全文下载
3. 隐藏自动化特征或规避风控
4. 保存账号密码
5. 遇到验证码后继续自动尝试

### 风控规则

1. 单次任务中文检索主题簇上限：10
2. 每个主题簇默认只查看前 1 页，必要时允许用户触发第 2 页
3. 相邻自动操作必须带间隔，不做高速连续点击
4. 每个主题簇在用户确认前不继续下一主题
5. 检测到验证码、登录失效、访问受限时立即中止

## 后端模块设计

建议在 `scripts/` 下新增独立模块，而不是复用 `run_round` 体系。

### 1. `scripts/reference_pipeline.py`

主编排模块，负责整条参考文献工作流。

职责：

- 驱动分析、检索、绑定、导出各阶段
- 管理任务状态
- 组织中英文候选合并
- 产出最终引用结果

### 2. `scripts/reference_models.py`

定义本功能自己的数据结构。

建议包含：

- `ReferenceJob`
- `SentenceCandidate`
- `TopicCluster`
- `ReferenceCandidate`
- `CitationBinding`
- `ReferenceDraft`

### 3. `scripts/reference_document.py`

文档结构分析模块。

职责：

- 解析 `txt/docx`
- 切分段落和句子
- 识别标题、章节、已有参考文献区
- 输出结构化正文模型

### 4. `scripts/reference_analysis.py`

全文分析和条数估算模块。

职责：

- 句级引文需求打分
- 高引文需求句筛选
- 主题聚类
- 建议中英文文献数量
- 生成中文和英文检索词

### 5. `scripts/reference_search_english.py`

英文文献检索模块。

职责：

- 调用 `OpenAlex`
- 调用 `Crossref`
- 去重、校验、补全元数据
- 返回英文候选文献池

### 6. `scripts/reference_search_cn.py`

中文文献结果接收和整理模块。

职责：

- 接收 Browser MCP 辅助检索后的候选结果
- 规范化字段
- 标记用户确认状态
- 输出中文候选文献池

### 7. `scripts/reference_binding.py`

句级绑定模块。

职责：

- 将候选文献映射到高分句
- 控制同段引用密度
- 支持一文多句复用
- 生成最终 `[x]` 编号顺序

### 8. `scripts/reference_export.py`

引用写回和导出模块。

职责：

- 按句子位置插入 `[x]`
- 在文末生成参考文献区
- 生成最终 `txt/docx`

### 9. `scripts/reference_records.py`

独立记录模块。

职责：

- 保存参考文献任务历史
- 区分于现有降 AI 记录
- 记录候选文献、确认结果、导出文件路径

## Flask API 设计

建议在 [scripts/web_app.py](D:/code/new/baibaiAIGC/scripts/web_app.py) 中新增独立接口组。

### 文档与任务

- `POST /api/reference/upload-document`
  - 上传并创建引用任务

- `GET /api/reference/status`
  - 返回任务状态、分析阶段、检索阶段、导出阶段

- `GET /api/reference/history`
  - 返回历史任务列表

### 分析阶段

- `POST /api/reference/analyze`
  - 执行正文分析，输出候选句、主题簇、建议条数

- `POST /api/reference/configure`
  - 用户提交目标中文/英文条数

### 英文检索阶段

- `POST /api/reference/search-english`
  - 按主题簇检索英文候选

### 中文知网阶段

- `POST /api/reference/start-cn-browser-session`
  - 初始化知网辅助检索会话

- `POST /api/reference/submit-cn-candidates`
  - 保存用户在知网确认的中文候选文献

### 绑定与导出阶段

- `POST /api/reference/generate-bindings`
  - 基于已确认候选生成句级绑定建议

- `POST /api/reference/apply`
  - 正式插入 `[x]` 并生成参考文献区

- `GET /api/reference/preview`
  - 返回带引用预览

- `GET /api/reference/export`
  - 导出结果 `txt/docx`

## 前端页面设计

建议在现有工作台外，新增一个独立的“参考文献”页签或独立页面，不与当前 round 视图混在一起。

### 页面一：参考文献工作台

功能：

- 上传 `txt/docx`
- 显示文档信息
- 启动“分析全文”
- 显示建议总条数、建议中文/英文数
- 允许用户调整中文/英文目标数

核心信息区：

- 文档名
- 字数
- 识别到的章节
- 建议引用位置数量
- 建议中文/英文文献数

### 页面二：英文候选文献面板

功能：

- 查看每个主题簇的英文候选
- 展示标题、作者、年份、来源、DOI、摘要片段
- 显示是否通过校验

交互：

- 默认自动选中高置信候选
- 用户可取消不合适条目

### 页面三：知网辅助检索面板

功能：

- 显示当前主题簇和建议搜索词
- 引导用户登录知网
- 显示当前检索进度和安全提示
- 显示已确认的中文候选文献

必要提示：

- 仅在合法登录权限下使用
- 不保存账号密码
- 检测到验证码或风险页会停止

### 页面四：句级绑定预览页

功能：

- 左侧显示正文句子列表
- 右侧显示候选文献编号与对应关系
- 高亮显示将被插入引用的句子
- 支持查看某条文献被绑定到了哪些句子

交互：

- 默认采用系统绑定建议
- 第一版不要求用户逐句编辑，但可以查看结果

### 页面五：导出预览页

功能：

- 预览正文中 `[x]` 插入后的效果
- 预览文末参考文献区
- 导出 `docx/txt`

## 测试策略

1. 文档解析测试
   - `docx/txt` 解析
   - 参考文献区识别
   - 句子切分

2. 分析测试
   - 引文需求打分
   - 主题聚类
   - 条数估算

3. 英文检索测试
   - OpenAlex 结果解析
   - Crossref 校验
   - 去重

4. 绑定测试
   - 一文多句复用
   - 同段密度控制
   - 编号顺序稳定

5. 导出测试
   - 句内 `[x]` 插入
   - 文末参考文献生成
   - `docx` 导出内容完整

6. 中文知网流程测试
   - 会话初始化
   - 人工确认候选保存
   - 中断与超时处理

## 上线前必须满足的门槛

1. 中文知网 Browser MCP 流程可用
2. 英文检索结果经过真实来源校验
3. 不保存知网账号密码
4. 有检索频率和步数限制
5. 检测验证码/异常页后立即停止
6. 导出结果只包含正文实际引用到的文献

## 实施建议

建议分为两个实现阶段，但两个阶段都属于同一上线版本范围：

### 阶段 A

- 后端独立数据模型
- 文档分析
- 英文检索
- 句级绑定
- 导出链路
- 前端工作台和预览页

### 阶段 B

- 知网 Browser MCP 交互页
- 中文候选确认
- 风控与中断逻辑
- 中英文候选合并
- 联调验收

这样可以先把核心算法和导出闭环做稳，再接中文检索交互与合规边界。
